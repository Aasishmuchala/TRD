Rewrite AuthGate.jsx to:
1. Check if the user is authenticated by reading localStorage.getItem('godsEyeApiKey')
2. If authenticated (key exists and is not empty), render children
3. If not authenticated, redirect to /welcome using useNavigate

The new AuthGate must be much simpler than the current one. The device-code OAuth flow (PROVIDERS, startLogin, startPolling, etc.) belongs on the Welcome page, not in a gate component. Remove it entirely from AuthGate.

New implementation:
```jsx
import { useEffect } from 'react'
import { useNavigate } from 'react-router-dom'

export default function AuthGate({ children }) {
  const navigate = useNavigate()

  useEffect(() => {
    const key = localStorage.getItem('godsEyeApiKey')
    if (!key) {
      navigate('/welcome', { replace: true })
    }
  }, [navigate])

  const key = localStorage.getItem('godsEyeApiKey')
  if (!key) {
    // Render nothing while redirecting — avoid flash of protected content
    return null
  }

  return children
}
```

Key requirements:
- AuthGate MUST use useNavigate (not window.location) for React Router compatibility
- AuthGate MUST render null (not a loading spinner or login form) while redirecting
- AuthGate MUST NOT contain any login UI, provider selection, device code flow, or polling logic — that all lives in Welcome.jsx
- AuthGate MUST be importable and work as a wrapper component: `<AuthGate>{children}</AuthGate>`
- The authentication check is simply: `localStorage.getItem('godsEyeApiKey')` is non-null and non-empty
- Mock mode is stored as 'mock-mode' in localStorage by Welcome.jsx — that satisfies the auth check

Note: Welcome.jsx already handles both API key entry (`localStorage.setItem('godsEyeApiKey', apiKey)`) and mock mode (`localStorage.setItem('godsEyeApiKey', 'mock-mode')`). AuthGate only needs to verify the key exists.
