**handleRun:** On form submit (prevent default), if !fromDate || !toDate return. Run BOTH modes concurrently using Promise.allSettled:

```js
const handleRun = async (e) => {
  e.preventDefault()
  if (!fromDate || !toDate) return
  setQuantLoading(true)
  setHybridLoading(true)
  setQuantError(null)
  setHybridError(null)

  const [quantSettled, hybridSettled] = await Promise.allSettled([
    apiClient.runQuantBacktest({ instrument, from_date: fromDate, to_date: toDate }),
    apiClient.runHybridBacktest({ instrument, from_date: fromDate, to_date: toDate }),
  ])

  setQuantLoading(false)
  setHybridLoading(false)

  if (quantSettled.status === 'fulfilled') setQuantResult(quantSettled.value)
  else setQuantError(quantSettled.reason?.message || 'Rules-only backtest failed')

  if (hybridSettled.status === 'fulfilled') setHybridResult(hybridSettled.value)
  else setHybridError(hybridSettled.reason?.message || 'Hybrid backtest failed')
}
```

**Status row** (when loading): two status badges side by side — "RULES ONLY: RUNNING..." (text-primary) and "HYBRID: RUNNING..." (text-[#a78bfa]), each in a terminal-card pill. Show only while respective loading state is true.

**Error row** (per mode, when error): small red terminal-card with "Rules-Only error: {quantError}" and "Hybrid error: {hybridError}" — shown independently.
