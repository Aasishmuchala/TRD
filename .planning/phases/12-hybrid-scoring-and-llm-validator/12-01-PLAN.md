  <behavior>
    Formula (HYB-01):
      hybrid_score = round(quant_score * 0.60 + agent_consensus_score * 0.40, 1)
      hybrid_score clamped to [0, 100]

    Agent consensus score derivation:
      For each agent in final_outputs: normalize direction to buy_weight (BUY/STRONG_BUY = 1.0,
      SELL/STRONG_SELL = -1.0, HOLD = 0.0), multiply by conviction, sum, divide by agent count.
      agent_consensus_score = max(0, min(100, abs(weighted_avg_conviction)))
      consensus_direction = "BUY" if weighted_avg > 0, "SELL" if < 0, "HOLD" if == 0.

    Direction lock rule (HYB-03):
      The quant direction is authoritative for the final direction.
      Validator CANNOT change direction. Only conviction and tradeable status are mutable.

    Tier after validation:
      hybrid_score > 70 AND verdict != "skip"  → "strong"
      hybrid_score >= 50 AND verdict != "skip" → "moderate"
      hybrid_score < 50 OR verdict == "skip"   → "skip"

    Validator verdict semantics (HYB-02, HYB-03):
      "confirm"  → conviction unchanged, tradeable = True
      "adjust"   → conviction = max(0, conviction - adjustment_amount); tradeable = True if conviction > 0
      "skip"     → tradeable = False; conviction set to 0

    Instrument hint (HYB-04):
      Follows quant instrument_hint when quant direction != HOLD.
      Falls back to consensus direction when quant is HOLD but consensus is not.

    Test cases (write in RED order, implement GREEN):
      1. Formula correct: quant=70, agents avg=60 → hybrid = 70*0.6 + 60*0.4 = 42+24 = 66.0
      2. Formula clamp: quant=100, agents=100 → hybrid = 100.0 (not > 100)
      3. Confirm verdict: hybrid_score=66, verdict=confirm → conviction unchanged, tradeable=True
      4. Adjust verdict: hybrid_score=66, conviction=66, adjust by 20 → conviction=46, tradeable=True, direction unchanged
      5. Adjust cannot flip: quant direction=BUY, agent consensus=SELL → final direction=BUY (quant wins)
      6. Skip verdict: hybrid_score=66, verdict=skip → tradeable=False, tier="skip", conviction=0
      7. HOLD quant + BUY agents: final direction=HOLD (quant wins even at HOLD)
      8. HybridResult has all 7 fields: direction, hybrid_score, quant_breakdown, agent_breakdown,
         validator_verdict, validator_reasoning, instrument_hint
  </behavior>

    ```python
    from dataclasses import dataclass, field
    from typing import Dict, List, Optional
    from app.engine.quant_signal_engine import QuantScoreResult
    from app.api.schemas import AgentResponse

    # Verdict literals
    VERDICT_CONFIRM = "confirm"
    VERDICT_ADJUST  = "adjust"
    VERDICT_SKIP    = "skip"

    @dataclass
    class ValidatorVerdict:
        verdict: str          # "confirm" | "adjust" | "skip"
        reasoning: str        # plain-language explanation
        adjustment_amount: float = 0.0   # only used for "adjust"; how many conviction pts to subtract

    @dataclass
    class AgentBreakdownEntry:
        direction: str
        conviction: float

    @dataclass
    class HybridResult:
        direction: str                    # Final direction (quant direction, locked)
        hybrid_score: float               # 0-100 fused score
        conviction: float                 # Post-validation conviction (may be reduced)
        tradeable: bool                   # False if skip verdict or score < 50
        tier: str                         # "strong" | "moderate" | "skip"
        instrument_hint: str              # From quant or fallback consensus
        quant_breakdown: Dict             # quant_score + quant factors (from QuantScoreResult)
        agent_breakdown: Dict[str, AgentBreakdownEntry]
        agent_consensus_score: float      # Derived agent consensus component (pre-fusion)
        validator_verdict: str            # "confirm" | "adjust" | "skip"
        validator_reasoning: str          # LLM or mock explanation

    class HybridScorer:
        @staticmethod
        def compute_agent_consensus(
            final_outputs: Dict[str, AgentResponse],
        ) -> tuple[float, str]:
            """Returns (consensus_score 0-100, consensus_direction BUY|SELL|HOLD)."""

        @staticmethod
        def fuse(
            quant_result: QuantScoreResult,
            final_outputs: Dict[str, AgentResponse],
            validator: ValidatorVerdict,
        ) -> HybridResult:
            """Main entry point. Pure fusion — no I/O."""
    ```

    Key implementation notes:
    - Direction weight mapping: BUY→+1.0, STRONG_BUY→+1.0, SELL→-1.0, STRONG_SELL→-1.0, HOLD→0.0
    - Direction lock: always use quant_result.direction as final direction (quant wins)
    - instrument_hint: use quant_result.instrument_hint if quant direction != HOLD; otherwise
      derive from consensus_direction using same NIFTY/BANKNIFTY logic
    - quant_breakdown dict: {"score": quant_result.total_score, "direction": quant_result.direction,
      "tier": quant_result.tier, "factors": quant_result.factors}
    - agent_breakdown dict: {agent_key: {"direction": r.direction, "conviction": r.conviction}
      for agent_key, r in final_outputs.items()}
