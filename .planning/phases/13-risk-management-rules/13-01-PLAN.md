<feature>
  <name>RiskManager — position sizing + stop/target computation</name>
  <files>gods-eye/backend/app/engine/risk_manager.py, gods-eye/backend/tests/test_risk_manager.py</files>
  <behavior>
    RiskManager.compute(tier, conviction, entry_close, vix) → RiskParams

    Position sizing (RISK-02):
    - tier == "strong"   (score > 70)  → lots = 2
    - tier == "moderate" (score 50-70) → lots = 1
    - tier == "skip"                   → lots = 0, stop_distance=0, target_distance=0, stop_level=entry_close, target_level=entry_close

    Stop loss (RISK-03): stop_distance = round(vix * 5.0, 1)
    - VIX=14   → stop_distance = 70.0 pts
    - VIX=20   → stop_distance = 100.0 pts
    - VIX=25   → stop_distance = 125.0 pts

    Target exit (RISK-04): target_distance = round(stop_distance * 1.5, 1)
    - 70pt stop → 105pt target
    - 100pt stop → 150pt target

    Absolute levels (based on direction from tier context — but RiskManager.compute does NOT know direction):
    - stop_level and target_level require direction. Use a separate helper:
      RiskManager.apply_levels(params, direction, entry_close) → (stop_level, target_level)
    - BUY: stop_level = entry_close - stop_distance; target_level = entry_close + target_distance
    - SELL: stop_level = entry_close + stop_distance; target_level = entry_close - target_distance
    - HOLD/skip: stop_level = entry_close; target_level = entry_close

    RiskParams dataclass fields:
      lots: int
      stop_distance: float   # points away from entry
      target_distance: float # points away from entry (always 1.5x stop_distance)
      stop_level: float      # absolute index level (entry_close ± stop_distance)
      target_level: float    # absolute index level (entry_close ± target_distance)
      vix_used: float        # VIX value used in computation (for response transparency)
      risk_reward: float     # always 1.5 (constant, shown for UI clarity)

    Simplification: compute() takes direction too (BUY/SELL/HOLD) so one call returns complete RiskParams.
    Signature: RiskManager.compute(tier: str, direction: str, entry_close: float, vix: float) -> RiskParams

    Edge cases:
    - vix <= 0 raises ValueError("VIX must be positive")
    - entry_close <= 0 raises ValueError("entry_close must be positive")
    - Unknown tier treated as "skip"

    Test cases (write these FIRST — RED phase):
    1. strong + BUY + close=22000 + VIX=20 → lots=2, stop_distance=100.0, target_distance=150.0, stop_level=21900.0, target_level=22150.0
    2. moderate + SELL + close=22000 + VIX=14 → lots=1, stop_distance=70.0, target_distance=105.0, stop_level=22070.0, target_level=21895.0
    3. skip + BUY + close=22000 + VIX=20 → lots=0, stop_distance=0.0, target_distance=0.0, stop_level=22000.0, target_level=22000.0
    4. strong + HOLD + close=22000 + VIX=20 → lots=2, stop_distance=100.0, stop_level=22000.0, target_level=22000.0 (HOLD = no directional levels)
    5. VIX=25 → stop_distance=125.0, target_distance=187.5
    6. vix=0 raises ValueError
    7. entry_close=0 raises ValueError
    8. risk_reward == 1.5 always (any valid inputs)
  </behavior>
