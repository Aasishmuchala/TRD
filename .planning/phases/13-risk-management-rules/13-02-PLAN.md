    **app/engine/daily_loss_guard.py** — new module, pure in-memory guard:

    ```
    DailyLossGuard tracks cumulative realised paper losses for the current trading day.
    "Day" = calendar date in Asia/Kolkata timezone. Resets automatically on new date.

    This is a paper trading guard — no real money, no DB persistence.
    State lives in the server process only; restarts reset it.
    ```

    Class: DailyLossGuard
    - __init__(self, max_daily_loss: float): stores limit, initialises _date=None, _cumulative_loss=0.0
    - _maybe_reset(self): if date.today(IST) != self._date, reset _cumulative_loss=0.0, _date=today
    - is_blocked(self) -> bool: call _maybe_reset(); return _cumulative_loss >= self.max_daily_loss
    - record_loss(self, loss_points: float) -> None: call _maybe_reset(); _cumulative_loss += max(0.0, loss_points)
    - cumulative_loss(self) -> float: call _maybe_reset(); return _cumulative_loss
    - reset(self) -> None: _cumulative_loss = 0.0; _date = today (used in tests / manual override)

    IST timezone: use `datetime.now(tz=timezone(timedelta(hours=5, minutes=30))).date()`

    Module-level singleton: `daily_loss_guard = DailyLossGuard(max_daily_loss=5000.0)`

    The singleton is initialised with the hardcoded default 5000.0. The route handler
    replaces it at startup if config.RISK_MAX_DAILY_LOSS differs (or just read from config
    at construction time — either is fine; simplest: use config at module load time).

    Simplest approach: `daily_loss_guard = DailyLossGuard(max_daily_loss=config.RISK_MAX_DAILY_LOSS)`
    Import config at top of daily_loss_guard.py.

    No locks needed — single worker gunicorn (established decision: [v1.0]).
