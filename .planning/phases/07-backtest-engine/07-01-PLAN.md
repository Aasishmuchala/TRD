<task type="auto" tdd="true">
  <name>Task 2: Implement BacktestEngine.run_backtest() with full day-loop logic</name>
  <files>gods-eye/backend/app/engine/backtest_engine.py</files>
  <behavior>
    - run_backtest(instrument, from_date, to_date, mock_mode=True) fetches all OHLCV rows for instrument and all VIX closes, then iterates each date in the fetched NIFTY rows that falls within [from_date, to_date]
    - For each signal day i, next day i+1 must also exist in the rows; if it doesn't (last day), skip it — cannot compute actual move
    - For each signal day: build MarketInput from historical row using _build_market_input(); set config.MOCK_MODE = mock_mode before calling orchestrator; call await orchestrator.run_simulation(market_input); compute consensus direction/conviction from final_outputs using equal-weight majority (most-voted direction wins; conviction = mean conviction of agents that voted the winning direction)
    - Compute actual_move_pct = (nifty_next_close - nifty_close) / nifty_close * 100; direction_correct per the semantics in Task 1; pnl_points per formula below
    - Per-agent accuracy aggregated at the end: for each agent, count days where that agent's direction matches direction_correct criteria (HOLD excluded from denominator)
    - overall_accuracy = sum(direction_correct) / len([d for d in days if d.direction_correct is not None])

    P&L formula (applied per day):
    ```
    if predicted_direction in ("BUY", "STRONG_BUY"):
        # Buy ATM CE — profits when market goes up
        # Proxy: option moves ~3x the underlying % move (simplified ATM CE delta ~0.5, leverage ~3x)
        option_leverage = 3.0
        pnl_points = actual_move_pct * option_leverage * (nifty_close / 100)
    elif predicted_direction in ("SELL", "STRONG_SELL"):
        # Buy ATM PE — profits when market goes down
        pnl_points = -actual_move_pct * option_leverage * (nifty_close / 100)
    else:  # HOLD
        pnl_points = 0.0
    ```
    (Result is in Nifty index points, representing approximate P&L on one ATM option lot)

    _build_market_input() uses historical row data:
    - nifty_spot = row["close"], nifty_open = row["open"], nifty_high = row["high"], nifty_low = row["low"], nifty_close = row["close"]
    - india_vix: look up VIX row for same date; default to 16.0 if missing
    - fii_flow_5d = 0.0 (not available historically — use neutral)
    - dii_flow_5d = 0.0
    - usd_inr = 84.0 (neutral proxy for historical dates)
    - dxy = 103.0 (neutral proxy)
    - pcr_index = 1.0 (neutral)
    - max_pain = row["close"]  (use close as proxy — no historical max pain)
    - dte = 15 (mid-series proxy)
    - rsi_14: compute from TechnicalSignals.compute_signals_for_date() using rows sliced to signal date
    - historical_prices: list of closes from all rows up to and including signal date (last 60 values to keep context manageable)
    - context = "normal"

    mock_mode=True (default): Sets config.MOCK_MODE = True before running each day. Orchestrator's agents (which check config.MOCK_MODE internally) will use MockResponseGenerator instead of calling the LLM API. Reset to original value after backtest completes.
  </behavior>
