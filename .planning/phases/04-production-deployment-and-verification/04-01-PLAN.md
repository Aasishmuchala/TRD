  <action>
    Make two targeted changes to Dockerfile:
    1. Change `-w 4` to `-w 1` in the CMD line. The StreamingOrchestrator and config are module-level singletons; multiple workers corrupt state. This is a known blocker recorded in STATE.md.
    2. Change `RUN mkdir -p /app/data` to `RUN mkdir -p /app/data /app/skills`. Both directories need to exist at image build time so Railway can mount volumes onto them.

    Keep all other Dockerfile lines unchanged.

    Then create TRD/gods-eye/backend/railway.toml with this exact content:

    ```toml
    [build]
    builder = "dockerfile"
    dockerfilePath = "./Dockerfile"

    [deploy]
    startCommand = "gunicorn run:app -w 1 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:$PORT --timeout 120 --access-logfile -"
    healthcheckPath = "/api/health"
    healthcheckTimeout = 300
    restartPolicyType = "on_failure"
    restartPolicyMaxRetries = 3
    ```

    Notes on railway.toml:
    - `startCommand` overrides the Dockerfile CMD so Railway's injected $PORT is used (Railway assigns random port, not always 8000).
    - `-w 1` is repeated here as the authoritative override — both Dockerfile and railway.toml agree.
    - `healthcheckPath` lets Railway confirm the container is up before routing traffic.
    - `dockerfilePath` is relative to the directory where railway.toml lives (backend/).
  </action>
