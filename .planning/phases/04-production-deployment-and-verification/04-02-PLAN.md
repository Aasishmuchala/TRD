    const wsBase = import.meta.env.VITE_WS_BASE

    - VITE_WS_BASE should NOT include a trailing slash and should include the protocol (wss:// or ws://).

    Explanation of each field:
    - `rewrites`: Catch-all rule that sends all paths (including /welcome, /dashboard, /agents, /history, /paper-trading, /settings, /skills) to index.html. React Router then handles client-side routing. This is the standard SPA fix for Vercel.
    - `buildCommand`: "npm run build" runs `vite build` (from package.json scripts).
    - `outputDirectory`: "dist" is where Vite puts the built assets (confirmed in vite.config.js — no custom outDir set).
    - `framework`: "vite" hints to Vercel which preset optimizations to apply.

    Do NOT set `devCommand` or `installCommand` — Vercel auto-detects npm.
    Do NOT add any API routes — the backend is on Railway, not Vercel serverless functions.
