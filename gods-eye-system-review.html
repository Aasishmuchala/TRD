Bottom Line

God's Eye has strong architecture — the 6-agent hybrid quant/LLM framework with 3-round interaction is well-designed and avoids single-model overfit. The quant engine is deterministic and reproducible. LLM integration via OpusCode Pro is working.

But the system has a critical gap in data quality: historical FII/DII flows and PCR are synthetic (momentum proxies), which means backtest results don't reflect real market dynamics. LLM agents operate without quantitative signal grounding (enriched_context pipeline is empty). And the HOLD-bias in prompts/aggregation needed multiple rounds of fixing.

The path to profitability: Fix data inputs (real flows, real PCR), ground LLM decisions in quant evidence, validate with 90-day live comparison, then optimize agent weights via feedback engine.

What's Working Well

● Multi-Agent Consensus

6 specialized agents (FII, DII, Retail F&O, Algo, Promoter, RBI) with domain-specific prompts. 3-round interaction allows agents to adjust after seeing others' views. Round 3 triggers only on direction change — avoids unnecessary LLM calls.

Production Quality

● Hybrid Aggregation (45/55)

Quant engine (45% weight) provides deterministic, reproducible base signal. LLM agents (55%) add contextual reasoning. Agreement boost (1.15x) rewards alignment; disagreement penalty (0.70x) flags conflict. This prevents either side from dominating.

Well Calibrated

● Quant Signal Engine

Deterministic scoring: FII/DII flows (±1000 Cr), PCR reversals (1.2/0.7), RSI extremes (<30/>70), VIX regime, Supertrend confirmation. Tier system (strong >70, moderate ≥50, skip <50) gates signal quality. Fully reproducible.

Solid

● Backtest Safeguards

signal_date windowing prevents future data leakage. Single-round parallel execution trades multi-round depth for speed — acceptable for historical validation. P&L with 3x leverage matches real derivatives exposure.

● Dhan API Integration

Accuracy-driven weight tuning (min 30 predictions, ±20% caps). Failure pattern detection: overconfident_buy, overconfident_sell, context_weakness. Corrective prompts inject past accuracy into agent decision-making.

Issues & Gaps — Prioritized

Historical FII/DII Flows Are Synthetic
Backtest derives flows from momentum proxy: fii = momentum_5d × 1500 Cr. Assumes perfect FII/DII negative correlation, ignores seasonal patterns (quarter-end rebalancing), flow convexity in stressed markets. Backtest P&L is plausible but not grounded in real market flows.
Critical Impact: Backtest results unreliable Fix: Integrate real NSDL/BSE flow data
Enriched Context Pipeline is Empty
The enriched_context parameter flows through to all agents but is always None. LLM agents see market numbers but not pre-computed quant signals, past accuracy stats, or knowledge graph context. They're flying blind on quantitative evidence.
Critical Impact: LLM decisions ungrounded Fix: Wire ProfileGenerator output → agent prompts
PCR Simulation From RSI
PCR = 0.6 + (RSI/100) × 0.8 — derives options sentiment from momentum. Misses expiry effects (weekly vs monthly), corporate events, hedging flows. Synthetic PCR is smoother and less predictive than real options data.
Critical Impact: Options signal unreliable in backtest Fix: Store real PCR from Dhan options chain daily
No Circuit Breaker for Dhan Outages
When Dhan returns None (timeout, 429, malformed), system logs a warning but may continue with stale cached data or null values. No explicit data validation before running simulation. Risk of silent degradation.
High Impact: Stale data decisions Fix: Add data freshness check + circuit breaker
Conviction Treats QUANT = LLM
Aggregator averages conviction across all agents without distinguishing reproducible QUANT signals (consistent across runs) from noisy LLM signals (sampled from 3 runs). A QUANT conviction of 80 is fundamentally more reliable than an LLM conviction of 80.
High Impact: Signal noise in consensus Fix: Weight by internal_consistency or reproducible flag
HOLD Bias in System (Partially Fixed)
Narrowed aggregator band (-8/+8), lowered consensus threshold (0.17), added decisive prompts, removed VIX hard override. But underlying issue remains: LLM models naturally hedge. Need ongoing calibration and A/B testing of prompt aggressiveness.
High Impact: Indecisive signals Fix: Monitor HOLD% and tune prompts quarterly
Round 3 Trigger is Fragile
Triggers only on direction string change. Misses conviction swings (BUY@30 → BUY@80 = "no change") and doesn't consider conflict magnitude. No configurable sensitivity.
Medium Impact: Missed re-evaluation signals Fix: Add conviction-delta trigger (>30pt)
Feedback Engine Needs 30 Predictions
Cold start problem: weight tuning doesn't activate until 30 simulations complete. No adaptive threshold. No decay of old predictions — 25 good + 5 bad treated same as random.
Medium Impact: Slow learning Fix: Lower to 15, add exponential decay
No Timezone Handling
Market data uses UTC timestamps but IST (UTC+5:30) is the operating timezone. Technical signals compare date strings without timezone awareness. Could cause edge-case bugs around market open/close.
Low Impact: Edge-case timing bugs Fix: Explicit IST conversion everywhere
